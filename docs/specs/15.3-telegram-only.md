# Spec: 15.3 — Telegram Stars Payment + Referral Cleanup

## Intent (for non-coders)

대표님 결정에 따라:
1. 외부 InviteMember 결제 링크 전량 제거 → Telegram Stars 자체 결제 시스템으로 통합
2. 래퍼럴(referral) 시스템 완전 삭제 (코드 잔재 제거, 기존 referred_by 데이터 보존)
3. 웹 랜딩 페이지(`apps/landing/**`) 디커미션
4. Premium 구독 가격 통일: **$39.99/월** (≈ 3,099 Telegram Stars)
5. Recharge 가격: **$6.50** (≈ 500 Telegram Stars)
6. 구독 갱신: **자동 30일 갱신** (Telegram `subscription_period=2592000`)
7. 청산 후 Recharge: wallet.balance += $1,000, is_liquidated=false, **seeded_at 보존** (시드 재발급 금지)
8. InviteMember 채팅방 → **봇 DM + broadcast 채널** 로 대체 (polling cron 제거)

## User Stories

- **구독자**: Premium 구독 시 Telegram Stars 결제 → 30일 자동 갱신 → premium_until 연장
- **청산 유저**: Recharge 버튼(LiquidationOverlay) → Stars 결제 → wallet.balance $1,000 충전 → 재매매 가능
- **구독 만료**: premium_until < now() → is_premium=false → 분석 모듈 자동 잠김 (매매 자체는 계속)
- **비프리미엄**: `/api/premium/analytics` 항상 401 or locked state 반환

## Chunks

1. **Data Migration & Schema** — Supabase 스키마 정리 (referred_by drop, referral_missions 제거, subscription 추가) — supabase-specialist — ~30min
2. **Telegram Stars Backend** — bot/src/engine/payment.ts 신규 (invoice 생성/성공 핸들러) + bot/src/index.ts 급여금 hook — frontend-developer — ~40min
3. **Liquidation Recharge Flow** — LiquidationOverlay → tg.openInvoice() → successful_payment hook → wallet 충전 — frontend-developer — ~30min
4. **InviteMember 폐기** — VITE_INVITEMEMBER_* 환경변수 제거, PremiumTab 링크 해제, bot polling cron 삭제 — frontend-developer — ~20min
5. **Referral Cleanup** — bot/src/engine/trading.ts 의 grantReferralBonus 호출 제거, bot/src/bot.ts /start 래퍼럴 분기 제거 — frontend-developer — ~20min
6. **Landing Page Archive** — apps/landing/** git rm (선택: 단일 redirect 페이지 or 완전 제거) — frontend-developer — ~15min
7. **Verification & E2E** — typecheck/lint + browser smoke test (subscribe → invoice → payment success → premium_until 갱신) — verifier-runner + browser-verifier — ~40min

## Data Shape

### users 테이블 변경
```sql
-- DROP
ALTER TABLE users DROP COLUMN IF EXISTS referred_by;  -- 레퍼럴 역사 삭제

-- ADD (신규)
ALTER TABLE users
  ADD COLUMN IF NOT EXISTS subscription_id TEXT UNIQUE DEFAULT NULL,  -- Telegram Bot API subscription handle
  ADD COLUMN IF NOT EXISTS premium_until TIMESTAMPTZ DEFAULT NULL,
  ADD COLUMN IF NOT EXISTS is_premium BOOLEAN DEFAULT false;

-- INDEX
CREATE INDEX IF NOT EXISTS idx_users_premium ON users(is_premium) WHERE is_premium = true;
CREATE INDEX IF NOT EXISTS idx_users_premium_until ON users(premium_until) WHERE premium_until > now();
```

### subscription_txns (신규 테이블)
```sql
CREATE TABLE IF NOT EXISTS public.subscription_txns (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES public.users(id) ON DELETE CASCADE,
  subscription_id TEXT NOT NULL,  -- Telegram Bot API response.subscription_id
  amount_usd NUMERIC(10, 2) NOT NULL,  -- 39.99 or 500 stars equiv
  currency TEXT DEFAULT 'stars',
  status TEXT NOT NULL,  -- 'pending' | 'active' | 'failed' | 'expired'
  period_start TIMESTAMPTZ DEFAULT now(),
  period_end TIMESTAMPTZ NOT NULL,
  created_at TIMESTAMPTZ DEFAULT now(),
  UNIQUE(user_id, subscription_id)
);

CREATE INDEX idx_subscription_txns_user ON subscription_txns(user_id);
CREATE INDEX idx_subscription_txns_status ON subscription_txns(status) WHERE status = 'active';
```

### recharge_txns (신규 테이블)
```sql
CREATE TABLE IF NOT EXISTS public.recharge_txns (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES public.users(id) ON DELETE CASCADE,
  amount_usd NUMERIC(10, 2) NOT NULL,
  currency TEXT DEFAULT 'stars',
  status TEXT NOT NULL,  -- 'pending' | 'completed' | 'failed'
  created_at TIMESTAMPTZ DEFAULT now(),
  completed_at TIMESTAMPTZ DEFAULT NULL
);

CREATE INDEX idx_recharge_txns_user ON recharge_txns(user_id);
```

### DROP (삭제)
- `referral_missions` 테이블 전체
- `users.referred_by` 컬럼

## Error States

- **Stars 부족**: `tg.openInvoice()` 호출 시 유저가 Stars 부족 → Telegram 기본 UI (구매 프롬프트)
- **결제 취소**: `pre_checkout_query` → false 반환 (Telegram 기본) or 타임아웃 → wallet 변화 없음, premium_until 그대로
- **중복 결제**: subscription_id 중복 → UNIQUE constraint + idempotent key (Telegram paid_media_payload)
- **시드 재발급**: seeded_at 이미 set → grantInitialSeed() 무시, balance 기존값 유지
- **구독 만료**: premium_until < now() 체크를 cron (매분) 또는 API 응답 시점에 실행 → is_premium flip false
- **네트워크 실패**: successful_payment 미수신 → user cron (`syncPremiumStatus.ts`) 폴백 재확인 (최대 1시간)

## Data Flow Diagram

```
[Premium Subscribe] — 사용자가 PremiumTab 에서 "Upgrade" 클릭
  ↓
TG.openInvoice({
  title: "Premium Membership",
  description: "30-day access",
  currency: "XTR",  // Telegram Stars
  prices: [{ amount: 3099 }]  // 3099 ⭐ ≈ $39.99
})
  ↓
[Telegram Payment UI] — 유저 Stars 결제
  ↓
bot.on('pre_checkout_query', ctx => ctx.answerPreCheckoutQuery(true))  // 승인
  ↓
bot.on('successful_payment', ctx => {
  subscription_id = ctx.msg.successful_payment.telegram_payment_charge_id
  premium_until = now() + 30 days
  users.update({ is_premium: true, premium_until, subscription_id })
  subscription_txns.insert({ status: 'active', period_end: premium_until })
  ctx.reply("✅ Premium activated for 30 days")
})

---

[Recharge After Liquidation]
  ↓
LiquidationOverlay "Risk Reset $6.50 / 500 ⭐" → onClick → tg.openInvoice({ amount: 500 })
  ↓
[Telegram Payment UI] — 유저 Stars 결제
  ↓
successful_payment → recharge_txns.insert({ status: 'completed' })
  ↓
wallets.update({ balance = balance + 100_000 (= $1,000) })
  ↓
/api/trade/{open,close} 재개 가능
```
