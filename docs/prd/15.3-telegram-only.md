# PRD: 15.3 — Telegram Stars Payment + Legacy Cleanup

## Business Intent

**Goal**: Simplify payment infrastructure by consolidating all transactions (Premium subscription + Recharge after liquidation) into native Telegram Stars, eliminating external InviteMember dependency and orphaned referral code.

**Why Now**: 
- InviteMember adds complexity & UX friction (external redirect → session loss on some clients)
- Referral system (3-refer/$50K, 10-refer/promo) was never fully utilized — low engagement
- Telegram Stars native integration is simpler, lower latency, and aligns with Telegram's official roadmap
- Landing page (`apps/landing/**`) is deployment overhead with minimal ROI

**Success Metrics**:
- Premium subscription conversion: ≥8% of DAU (currently ~3% via InviteMember)
- Recharge after liquidation: ≥60% of liquidated users recover within 24h
- Total users w/ active subscription: ≥5% by Q3 2026
- Code cleanup: 0 referral-related entries in codebase (grep confirmed)

## Scope: What Gets Built

### 1. Telegram Stars Billing
- Native Telegram Bot API `sendInvoice()` + `pre_checkout_query` + `successful_payment`
- Premium: $39.99/month → 3,099 ⭐ (auto-renews every 30 days)
- Recharge: $6.50 (risk reset) → 500 ⭐
- Idempotent: duplicate `telegram_payment_charge_id` ignored (no double charge)

### 2. Premium Features Gate
- `/api/premium/analytics` returns 403 if `is_premium=false` or `premium_until < now()`
- Modules B/C/D (hourly buckets, leverage, behavior) auto-lock with UI hint "Upgrade to Premium"
- Module A (trade stats) remains free
- Lock timeout checked on every API call + nightly cron

### 3. Recharge Flow
- LiquidationOverlay shows "Risk Reset" button (not "Recharge")
- Click → `tg.openInvoice(500⭐)` → successful_payment → wallet += $1,000
- `is_liquidated` flip false (trade resume)
- `seeded_at` preserved (no fresh seed)

### 4. Cleanup
- Remove referral_missions table
- Drop users.referred_by column
- Kill InviteMember polling cron + env vars
- Archive landing page (or minimal 1-pager redirect)
- Remove bot.ts `/start` referrer logic

## Success Criteria (Acceptance)

1. **Premium Subscribe**: User taps "Unlock Premium" → Telegram invoice (Stars) → payment succeeds → `is_premium=true`, `premium_until=now()+30d` in DB
2. **Auto-Renew**: On day 30, Telegram auto-deducts 3,099⭐ → `premium_until` rolls forward +30d (or user sees renewal prompt)
3. **Recharge**: Liquidated user taps "Reset Risk" → 500⭐ invoice → success → wallet.balance += 1000, can trade again
4. **Premium Lock**: Logout & re-login with non-premium account → PremiumTab modules B/C/D show locked overlay
5. **Referral Removal**: `grep -r "referred_by\|referral_missions\|grantReferralBonus"` across bot/ + web/ → 0 matches

## Metrics & Analytics

- **Daily Premium Signups** (segment: is_premium=true, created_at > now()-24h)
- **Subscription Retention** (30d / 60d)
- **Recharge Rate** (attempts / liquidations)
- **Star Exchange Efficiency** (avg USD → ⭐ actual payout vs list price)

## Non-Goals / Out of Scope

- Telegram Stars withdrawal/payout (partner only)
- Multi-tier subscriptions (Pro/Elite tiers)
- Trial periods or freemium downgrades
- Regional pricing
- Subscription pause/resume

## Risks & Mitigations

| Risk | Impact | Mitigation |
|------|--------|-----------|
| Telegram API outage | No new subscriptions | Graceful fallback: show last-known state for 24h |
| User forgets to renew (no auto-deduct) | Churn | Send reminder DM 3 days before expiry |
| Liquidated user can't afford $6.50 reset | Lost engagement | Offer "free reset" once per 7 days (discount tier) |
| Referral bonus recipients feel cheated | Reputation | 1-time $10K gift for all existing referral-bonus users (sunk cost, no migration needed) |
